## 2026/04/20 用户中心与 VIP 升级会员按钮
### 如何测试
1. **社区版测试**：设置 `is_community=true`
   - 标题栏显示登录图标（无文字），点击直接跳转商业版官网
   - 不应显示 VIP 按钮
2. **商业版未登录状态**：设置 `is_community=false`
   - 启动后应看到蓝色"未登录"文字 + 登录图标（日间 `#0078d4`，夜间 `#4fc3f7`）
   - 登录按钮右侧应显示金色"升级会员"按钮
   - 点击登录按钮打开登录弹窗，点击 VIP 按钮跳转到会员购买页面
3. **商业版普通用户登录**：使用 "Regular User" 或 "Trial Member" 账号登录
   - 登录按钮应只显示图标，不显示用户名
   - VIP 按钮应保持显示
4. **商业版付费会员登录**：使用 "Fruit User"/"Sprout User"/"Seed User"/"Member" 账号登录
   - 登录按钮只显示图标
   - VIP 按钮应隐藏
5. **主题切换测试**：分别测试日间主题和夜间主题，检查按钮颜色是否正确

### What
1. **社区版简化逻辑**：社区版只显示登录图标（无文字），点击直接跳转官网，不显示 VIP 按钮
2. **商业版登录按钮功能**：
   - 未登录状态：显示蓝色"未登录"文字 + 图标
   - 已登录状态：只显示图标，不显示用户名
   - 按钮padding调整：左padding从10px减到6px，使文字不溢出
3. **商业版 VIP 按钮功能**：
   - 按钮位置：位于登录按钮右侧
   - 按钮样式：金色/橙色渐变背景（`#FFD700` 到 `#FFA500`），圆角设计，左侧闪电图标
   - 显示逻辑：未登录用户、普通用户、体验会员显示；付费会员隐藏
   - 点击行为：跳转到会员购买页面

### How
- `qt_tm_widget.cpp`:
  - 登录按钮创建包裹在 `if (!is_community_stem ())` 判断中
  - 新增 `updateLoginButtonState()` 方法，根据登录状态更新按钮显示
  - 登录状态通过 `login-state` 属性（`logged-in`/`not-logged-in`）传递给QSS
  - 创建 `QPushButton` 类型的 VIP 按钮，设置金色渐变样式
  - 实现 `updateVipButtonVisibility()` 方法控制 VIP 按钮显示/隐藏逻辑
  - 在 `updateDialogContent()` 中保存会员类型并更新 VIP 按钮可见性
- `windowbar.hpp` / `windowbar.cpp` / `windowbar_p.hpp`:
  - 在枚举中添加 `VipButton` 位置
  - 添加 `vipButton()` / `setVipButton()` / `takeVipButton()` 方法
- `liii.css` / `liii-night.css`:
  - 基础样式添加 `padding: 0 10px 0 6px` 和 `margin-right: 8px`
  - 通过 `[login-state="not-logged-in"]` 和 `[login-state="logged-in"]` 区分状态颜色
- `styles.qrc`: 添加闪电图标资源文件
- `zh_CN.scm`: 添加 "Upgrade VIP" -> "升级会员" 翻译

## 具体实现
### 涉及文件
- `TeXmacs/plugins/account/progs/liii/account.scm` - 通过scheme处理token缓存

- `fetchUserInfo()` - 获取用户信息（支持模拟数据）
- `triggerOAuth2()` - 触发OAuth2认证流程
- `updateDialogContent()` - 更新对话框内容
