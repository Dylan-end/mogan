#### 情况1：`NotificationBar` 显示 `update`

##### 步骤1：设置 Mock 版本
1. 因为测试版本一般都是最新的，测试版本号比远程获取的大，所以需要模拟一个更大的远程版本，在 Scheme 会话中执行：

3. 点击 `Remind later` / `稍后提醒`，看 update 提示是否消失

1. 重启应用，等待 10 秒后查看是否无 update 提示显示

2. 可选：重新触发 update 提示后，点击 `Skip this version` / `跳过此版本`，确认提示消失

## 2026年4月15日

### What
新增统一的顶部 SCM 通知条，并将版本更新、访客登录、会员临期续费、会员过期四类提示状态同步到这一套通知条逻辑中。

### 功能特性
1. **新增 SCM 顶部通知条通路** - 从 Scheme 菜单生成通知条内容，经 `SLOT_NOTIFICATION_BAR` 传递到 Qt 窗口显示
2. **多通知项轮播** - 根据当前激活状态，在版本更新、访客提示、会员临期提示、会员过期提示之间按 5 秒轮播
3. **状态同步** - Qt 原生提示条与 SCM 通知条共用同一份业务状态
4. **忽略指定版本** - 版本更新按钮从“稍后提醒”改为“忽略当前版本”
5. **会员提示** - 登录后后台刷新会员信息，在 3 天内到期时展示提前续费提醒，过期时展示查看会员方案入口

### 提示显示判定逻辑

SCM 顶部通知条会先收集当前所有激活项，再按数量决定是否轮播：

```text
Qt/Scheme 更新状态
    ↓
收集激活项：
  - update
  - guest
  - membership-renew-soon
  - membership
    ↓
count = 激活项数量
    ↓
count <= 1  ──→ 固定显示当前项
count > 1   ──→ 每5秒按 index 轮播
```

**各提示项激活条件：**

| 提示项 | 激活条件 |
|--------|----------|
| `update` | `updateAvailable = true` 且远程版本非空，且该版本未被忽略 |
| `guest` | 商业版、未登录、网络可用 |
| `membership-renew-soon` | 已登录、非 guest、有会员数据、不是免费用户，且 `periodLabel` 可解析为实际到期日期，并满足 `0 <= 到期日 - 今天 <= 3`，同时未处于 1 天静默期 |
| `membership` | 已登录、非 guest、有会员数据、不是免费用户，且 `periodLabel` 可解析为实际到期日期，并满足 `到期日 < 今天`；若日期无法解析，则兼容回退到旧的 `productType != "Renew Early"` 逻辑，同时未处于 7 天静默期 |

**免费用户判定：**

```text
memberType == "Regular User"
periodLabel == "Non-member"
```

### UI 行为
1. **版本更新提示**
   - `Update now`：打开下载页
   - `Remind later`：记录 `version_snooze_until`，3 天后再提示
   - `Skip this version`：记录当前远程版本到 `version_ignored_remote`
2. **访客提示**
   - `Login Now`：触发 OAuth2 登录
3. **会员临期提示**
   - 使用固定帮助式文案，不展示动态剩余天数
   - 主按钮固定为 `Renew Early`
   - 点击按钮或关闭 `×` 后静默 1 天
4. **会员过期提示**
   - 使用固定帮助式文案，不拼接 `periodLabel`
   - 主按钮固定为 `View plans`
   - 点击按钮或关闭 `×` 后静默 7 天

### Why
之前顶部通知能力分散在多个 Qt 提示条里，Scheme 菜单层没有统一的通知入口，也缺少多通知项轮播和版本忽略能力。这次把通知条链路打通后，后续新增提示类型会更容易扩展。

### How
1. 在窗口消息系统里新增 `SLOT_NOTIFICATION_BAR`，打通 `server -> tm_frame -> tm_window -> qt_tm_widget` 的通知条传输通路
2. 新增 `QWK::NotificationBar` 作为 SCM 菜单驱动的 Qt 外壳，用来承载菜单生成的 QLabel / QPushButton 内容
3. 在 `notificationbar.scm` 中维护四类通知状态，并通过 `notification-bar-active-items` 和 `notification-bar-current-item` 控制轮播
4. 在 `qt_tm_widget.cpp` 中将版本更新、访客状态、会员信息同步到 Scheme 层，同时继续兼容原有 Qt 提示条
5. 在 `version-update.scm` 中新增忽略指定远程版本的持久化存储，替代通知条中的“稍后提醒”按钮语义
6. 会员临期与过期提示分别使用独立静默 key，并优先基于 `periodLabel` 的实际日期计算；当日期格式无法解析时，再回退到旧的 `productType` 逻辑
